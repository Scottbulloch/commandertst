Commander Legacy API

The latest version of Commander currently supports the Commander Legacy API. However, new feature development is being done for the Commander API. Therefore, unless you're supporting existing applications, we encourage you to use Commander API.

To view the reference documentation for the Commander Legacy API in a new browser or tab, click the link below:

Commander Legacy API Reference Docs

If you have access to a Commander installation, you can enter the following URL in your browser to access the reference documentation for the Commander Legacy API in a separate browser window:
https://<Commander hostname or IP address>:<port>/apihelp

For additional information, you can also see the Commander API Getting Started Guide (included with the REST API package).

While you can use any REST client to communicate with the Commander Legacy API, Commander also provides a PowerShell client. This client converts the XML input/output parameters into PowerShell objects. To download the REST PowerShell client, log in to Snow Globe, then download the client from the Download Commander PowerShell Module for REST article.
