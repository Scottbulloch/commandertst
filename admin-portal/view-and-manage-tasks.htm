View and Manage Tasks

In Commander, a task represents an action taken by a user. Tasks are considered to be either current or historical:

  • Current tasks — Tasks that are no more than five minutes old. For quick reference, current tasks are displayed in a Tasks tab, which is always present at the bottom of the Admin Portal. You can also cancel some queued tasks before they start to run.
  • Historical tasks — Tasks of all ages. Historical tasks are displayed in another Tasks tab at the top right of the Admin Portal.

Your view of tasks is based on your access rights. If you don't have access rights to some infrastructure elements or VMs, you aren't allowed to see the tasks for those infrastructure elements or VMs.

View current tasks

Access:

Tasks tab at bottom of console

Available to:

All Access Rights Levels

  • To view the current tasks, select the Tasks tab at bottom of console.

 

  • The pane is minimized by default, but if you expand the tab, your setting is preserved between sessions.
  • Double-click a task to view more details.

Displayed task information

The following information is displayed for listed tasks. You can filter your search based on any combination of these fields.

  • Name — The name of the task. Typically this name tells you at-a-glance what the task is, for example, a deployment.
  • Source — The name of the entity on which the action is taken.
  • Destination — The destination of the action taken on the VM, such as a folder or host.
  • Status — The status of the task. It can be queued, running, or completed.
  • Initiated By — The name of the user who started the task.
  • Submitted Time — The time the task is placed in the queue for processing.
  • Start Time — The time the task starts.
  • Completed Time — The time the task finishes.

Cancel queued tasks

You can cancel the following tasks only while they are queued and before they start to run:

  • Datastore scans
  • Historical event retrieval
  • Run Program in Guest workflow steps
  • Upload Media File
  • Refresh Details for a media folder

Access:

Views > Inventory > Infrastructure, Applications, or Storage

Available to:

All access rights levels (users can cancel their own tasks)

Superuser role and Administrator access rights (users can cancel all tasks)

  1. Click the Tasks tab at the bottom of the Admin Portal.
  2. In the current Tasks tab, select the queued task you want to cancel.
  3. Right-click the queued task and select Cancel.
  4. Click Yes to confirm the cancellation.

View historical tasks

Access:

Views > Inventory > Infrastructure, Applications, or Storage

Available to:

All Access Rights Levels

  1. From the Inventory tree, select the level of your virtual infrastructure that you want to view historical tasks for.
  2. Click the Tasks tab in the upper section of the console.

    For descriptions of the fields displayed for tasks, see Displayed task information.

     

    • Because of the number of available tabs, you may have to click the > button to bring the tab into view.
    • If you want to view all the tasks that occurred in your virtual infrastructure, select the root element of the inventory tree.
