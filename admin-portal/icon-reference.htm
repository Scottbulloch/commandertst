Icon Reference

AWS icons

The following icons are displayed only for AWS cloud accounts.

Icon

Name

Description

Availability Zone

A datacenter. AWS availability zones are represented as datacenters at the cloud account level in Commander.

Auto Scaling Group

An EC2 Auto Scaling Group

Auto Scaling Group alert

An EC2 Auto Scaling Group indicating that the current number of instances falls outside the minimum and maximum capacity range, or that instances in the group are in the process of launching or terminating to reach desired capacity.

Cloud account

A connected cloud account.

Cloud account disconnected

A disconnected cloud account.

Database

An RDS database instance.

Database alert

An RDS database instance with an alert condition.

Database (Read Replica)

An RDS read replica.

Database (Read Replica) alert

An RDS read replica with an alert condition.

Datastore

A datastore.

EC2-Classic

A container to represent the parent of instances running in EC2-classic (as opposed to in a VPC)

Folder

A folder. For AWS, Commander uses folders to organize resources in the Inventory trees.

Load Balancer

An Elastic Load Balancer. The load balancer type is indicated on the load balancer summary page. It can be a Classic Load Balancer, an Application Load Balancer, or a Network Load Balancer.

Load Balancer alert

An Elastic Load Balancer with an alert condition. This may indicate that it's unapproved. It could also indicate that the load balancer has unhealthy instances associated with it or its target groups.

Region

An AWS region.

Stack

An AWS stack.

Stack alert

An AWS stack with an alert condition.

VM (powered on)

A VM that's powered on and in a running state.

VM (powered off)

A VM that isn't running.

VM alert

A VM with an alert condition.

VPC / Virtual Network

A Virtual Private Cloud.

Service request and workflow icons

The following icons can be displayed for steps in the Request Details dialog and Workflow Status page.

Icon

State

Description



Waiting/In Progress

A step is waiting for something to occur, for example, an approval. Or the step is in progress, for example, when deployment is in progress.



Error

An error occurred, for example, a failed step in the approval workflow, or a failed automated deployment. See also Troubleshooting Automated Deployment.



Attention

The step requires action; for example, a VM is ready for deployment.

Click Deploy at the service level or at the component level of the request details tree to launch the deployment wizard.

Mouse over steps requiring attention to see more details.



Rejected

The service request or approval workflow has been rejected.



Running

The workflow is running.



Completed

The workflow or workflow step has completed (or, in the case of a parent node, all child steps have completed).

Not Started

The step hasn't started.

Started

The step is in progress.
